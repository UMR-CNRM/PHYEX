PHYEX Documentation

Doxygen is used to document the source code, general documentation is available directly in the docs directory as Makdown files and are linked here:

  • Introduction
  • Developer: package organisation, how to contribute, coding norms
  • Coding norms: coding norms
  • Integrator: how to merge contributions
  • Offline: how to compile the library and the test programs, how to use the library with python and how to use the test programs
  • Plugging : how to plug the physics package in a model
  • Tools: description of the check_commit_*.sh scripts (to check bit reproducibility between two commits) and of the prep_code.sh script

The main entry points of the package are:

  • ini_phyex: to setup the different schemes
  • rain_ice: to run the ICE3/ICE4 microphysics scheme
  • rain_ice_old: to run the old version of ICE3/ICE4 microphysics scheme
  • ice_adjust: to run the adjustment process linked to the ICE3/ICE4 scheme
  • turb: to run the turbulence scheme
  • shallow_mf: to run the shallow convection scheme

In addition, some offline tools are available:

  • main_rain_ice: to drive the rain_ice subroutine
  • main_rain_ice_old: to drive the rain_ice_old subroutine
  • main_turb: to drive the turb subroutine
  • main_ice_adjust: to drive the ice_adjust subroutine
  • main_shallow: to drive the shallow_mf subroutine
